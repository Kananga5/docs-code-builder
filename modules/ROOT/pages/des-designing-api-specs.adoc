= Designing API Specifications
// :page-aliases: help-overview-design.adoc

include::reuse::partial$beta-banner.adoc[tag="anypoint-code-builder"] 

Use Anypoint Code Builder to create, design, test, synch with Design Center, and publish new RAML or OAS API specifications to Exchange.

The process for designing API specifications is:

. xref:des-create-api-specs.adoc[Design your API specification].
. xref:des-add-examples-to-specs.adoc[Optionally, add examples to your specification].
. xref:des-test-api-specs.adoc[Test your API specification using the Mocking Service].
. xref:des-sync-with-design-center.adoc[Synchronize your API specification with Design Center].
. xref:des-publish-api-spec-to-exchange.adoc[Publish your API specifications to Exchange].

[[supported-spec-languages]]
== Supported OAS and RAML Versions

Supported OAS and RAML versions are:

* RAML 1.0
* RAML 0.8
* OAS 2.0 (JSON)
* OAS 2.0 (YAML)
* OAS 3.0 (JSON)
* OAS 3.0 (YAML)
