Import a project directory or a packaged project into a workspace in your IDE. A project directory contains an API specification or a Mule application for an implementation or integration. 

Visual Studio Code treats a project directory as a workspace when you open it to the project's root directory or if your project folder is located in a https://code.visualstudio.com/docs/editor/multi-root-workspaces[multi-root workspace^]. 

== Import a Project into a Desktop IDE Workspace

The desktop IDE enables you to upload a project directory or package JAR file into a workspace. 

* <<desktop-project-folder>>
* <<desktop-project-package>>

[[desktop-project-folder]]
=== Import a Project Folder into a Desktop IDE Workspace

. Open the Command Palette by pressing Ctrl+Shift+p (Windows) or Cmd+Shift+p (Mac), and provide the following command:
+
[source,command]
----
File: Open Folder...
----
. Navigate to the _root_ directory for your project. 
+
// see text for this variable defined under title
{root-dir}
. Open the root directory for your project.
+
// see text for this variable defined under title
{open-root}

[[desktop-project-package]]
=== Import a Packaged Project into a Desktop IDE Workspace

Import a JAR file and automatically unpack the contents to a project folder.

. Open the Command Palette by pressing Ctrl+Shift+p (Windows) or Cmd+Shift+p (Mac), and provide the following command:
+
[source,command]
----
MuleSoft: Import a Project
----
. Select *Mule Project Type*:
+
image::import-mule-project.png[]
. Navigate to the JAR file and click *Select a jar file to import*.
. Navigate to a parent directory into which you want to store the decompressed project folder and click *Select a folder to work*.
+
The IDE unpacks the JAR file, loads the project folder, and opens the project workspace in the IDE.

[[use-dw-imported-project]]
== Use DataWeave with Imported Projects

To use DataWeave in an imported project, a `mule-service-weave-ee` dependency must be configured in your project `pom.xml` file, for example:

[source,XML]
----
<dependency>
  <groupId>com.mulesoft.services</groupId>
  <artifactId>mule-service-weave-ee</artifactId>
  <version>2.4.0-20220824</version>
  <scope>runtime</scope>
</dependency>
----
